# 连接生产服务器进行完整用户功能测试计划

## 测试目标
使用 Chrome DevTools MCP 工具访问生产服务器 (http://120.26.50.152)，按照《MCP 用户流程测试计划》执行 11 个主要模块的完整功能测试。

## 测试步骤

### 1. 准备阶段
- 使用 Chrome DevTools MCP 打开生产服务器 URL: http://120.26.50.152
- 验证页面加载正常
- 记录初始状态

### 2. 执行 11 个测试模块
根据 `.trae/documents/MCP 用户流程测试计划.md` 中的测试计划执行：

**模块 1：启动序列与认证系统**
- 观察启动动画效果
- 测试用户注册功能
- 测试用户登录功能
- 测试登出功能

**模块 2：仪表板功能**
- 验证系统监控图表显示
- 测试 AI 伴侣交互
- 测试快捷访问卡片
- 验证数据统计

**模块 3：笔记系统完整测试**
- 文件夹操作（创建、重命名、删除）
- 笔记创建和编辑
- 标签管理
- 笔记操作（置顶、删除、批量操作、搜索）
- 视图切换和专注模式

**模块 4：每日复盘系统**
- 今日复盘（日期、内容、情绪评分、专注度、能量等）
- 生物指标雷达图验证
- 历史记录查看和筛选
- 统计信息验证

**模块 5：系统配置**
- 用户信息显示
- 主题切换（Cyan/Pink/Yellow）
- 语言切换（中文/English/日本語）
- 界面密度、数据同步、带宽限制等设置
- 通知设置和安全设置

**模块 6：GitHub 项目监控**
- 访问 /github 路由
- 验证仓库信息和实时统计
- 查看提交历史和自动刷新

**模块 7：连接测试页面**
- 访问 /test-connection
- 运行完整的 API 测试套件（10 项测试）
- 验证测试结果摘要

**模块 8：备份系统**
- 创建备份
- 查看备份列表
- 测试备份详情和下载

**模块 9：响应式布局测试**
- 测试不同屏幕尺寸（移动端、平板、桌面）
- 验证侧边栏和图表适配

**模块 10：跨页面导航测试**
- 测试侧边栏导航
- 验证 Hash 路由更新
- 测试浏览器前进后退

**模块 11：错误处理测试**
- 网络错误处理
- 表单验证
- 权限检查

### 3. 测试记录
- 每个模块完成后截图
- 记录测试结果（成功/失败）
- 记录发现的任何问题或错误

### 4. 生成测试报告
- 汇总所有测试结果
- 列出发现的问题
- 提供测试结论和建议

## 工具使用
- 使用 `mcp_Chrome_DevTools_MCP_navigate_page` 访问页面
- 使用 `mcp_Chrome_DevTools_MCP_take_snapshot` 获取页面状态
- 使用 `mcp_Chrome_DevTools_MCP_click` 点击元素
- 使用 `mcp_Chrome_DevTools_MCP_fill` 填写表单
- 使用 `mcp_Chrome_DevTools_MCP_take_screenshot` 截图
