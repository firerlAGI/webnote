# Product Context - WebNote

## 产品愿景

**让笔记和复盘回归简单**

WebNote 旨在为用户提供一个极致简约的笔记管理和日复盘工具，专注于核心功能，避免功能膨胀和复杂操作，让用户能够专注于思考和记录本身。

## 存在的问题

### 当前市场痛点

1. **笔记工具过度复杂**
   - 功能繁多但实际使用率低
   - 学习成本高，新用户难以快速上手
   - 界面拥挤，干扰用户专注写作

2. **复盘工具匮乏**
   - 缺乏专门的日复盘工具
   - 数据可视化不友好
   - 无法长期追踪复盘趋势

3. **多设备同步复杂**
   - 分散工具同步功能不稳定
   - 冲突解决机制不友好
   - 离线体验差

## 目标用户

### 核心用户画像

**知识工作者**
- 年龄：25-40 岁
- 职业：程序员、产品经理、设计师、创业者
- 特征：
  - 注重效率和简洁性
  - 有每日复盘习惯
  - 使用多个设备（桌面、移动端）
  - 希望工具即插即用，无需复杂配置

### 用户需求

1. **核心功能需求**
   - 快速记录想法和笔记
   - Markdown 编辑支持
   - 每日复盘记录和统计
   - 多设备数据同步
   - 数据安全备份

2. **非功能需求**
   - 界面简洁美观
   - 操作响应快速
   - 数据安全可靠
   - 无需学习成本

## 产品价值主张

### 核心价值

1. **极简设计**

## 产品工作方式

### 用户旅程

#### 1. 首次使用

```
注册/登录
    ↓
欢迎引导（极简）
    ↓
创建第一篇笔记
    ↓
体验核心功能
```

#### 2. 日常使用

**笔记工作流**:
```
打开应用
    ↓
浏览笔记列表 / 搜索笔记
    ↓
创建/编辑笔记（Markdown 编辑器）
    ↓
自动保存 + 同步
    ↓
关闭应用
```

**复盘工作流**:
```
打开复盘页面
    ↓
记录今日复盘（情绪、完成度、反思）
    ↓
查看历史复盘统计
    ↓
分析趋势和改进点
```

### 核心功能流程

#### 笔记管理

1. **创建笔记**
   - 点击"新建笔记"按钮
   - 自动生成标题（可编辑）
   - 进入 Markdown 编辑器
   - 实时自动保存

2. **编辑笔记**
   - 实时预览
   - 工具栏快捷操作（加粗、斜体、代码块、图片上传等）
   - 快捷键支持

3. **管理笔记**
   - 搜索笔记（标题/内容）
   - 分类文件夹
   - 标签管理
   - 删除笔记

#### 复盘管理

1. **创建复盘**
   - 进入复盘页面
   - 选择日期
   - 记录情绪、完成度、反思
   - 提交保存

2. **查看统计**
   - 情绪趋势图表
   - 完成率分析
   - 历史复盘列表
   - 分页控件

#### 数据同步

1. **在线模式**
   - WebSocket 实时同步
   - 冲突自动解决（5 种策略）
   - 心跳保持连接

2. **离线模式**
   - 本地缓存（IndexedDB）
   - 三级缓存架构
   - 重新上线后自动同步

## 用户体验目标

### 设计原则

1. **简洁性优先**
   - 界面元素最小化
   - 避免视觉噪音
   - 功能聚焦

2. **响应快速**
   - 页面加载 < 2s
   - 交互响应 < 100ms
   - 数据同步流畅

3. **操作直观**
   - 无需学习成本
   - 常用操作一键可达
   - 清晰的反馈提示

4. **渐进增强**
   - 核心功能优先
   - 高级功能可选
   - 用户自主选择

### 界面风格

- **色彩**: 柔和中性色调，突出内容
- **布局**: 清晰的信息层次
- **字体**: 现代无衬线字体，易读性强
- **动画**: 微妙流畅，不干扰视线

### 交互设计

- **即时反馈**: 所有操作都有明确反馈
- **容错设计**: 重要操作确认提示
- **快捷键**: 提高熟练用户效率
- **手势支持**: 触摸设备友好
