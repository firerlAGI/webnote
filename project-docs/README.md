## 📋 文档分类

### 🎯 项目规划与设计

#### [会议记录.md](./会议记录.md)
- **描述**：项目初始会议记录，包含产品需求分析、技术方案设计、MVP规划等
- **适用人群**：所有团队成员
- **更新频率**：不常更新
- **关键内容**：
  - 产品核心需求（极简网页笔记 + 轻量化日复盘）
  - Monorepo架构设计
  - 技术栈选择

  - 开发与部署流程

- **适用人群**：架构师、开发者
- **更新频率**：架构变更时更新
- **关键内容**：
  - Monorepo结构设计
  - 前后端技术选型
  - 数据模型设计
  - API设计规范
  - 部署架构

### 📝 开发指南

#### [开发规范.md](./开发规范.md)
- **描述**：代码规范和最佳实践
- **适用人群**：所有开发者
- **更新频率**：规范变更时更新
- **关键内容**：
  - 代码风格规范
  - Git提交规范
  - 命名规范
  - 目录结构规范
  - 注释规范

#### [开发流程图.md](./开发流程图.md)
- **描述**：开发工作流程和协作流程
- **适用人群**：所有团队成员
- **更新频率**：流程优化时更新
- **关键内容**：
  - 分支管理策略
  - 代码审查流程
  - CI/CD流程
  - 发布流程

### 📊 项目进度

#### [项目进度追踪.md](./项目进度追踪.md)
- **描述**：实时项目进度追踪，包含任务状态、里程碑、风险等

## 📌 文档维护指南

### 文档更新责任

| 文档类别 | 负责人 | 更新频率 |
|---------|--------|---------|
| 项目规划文档 | 项目负责人 | 需求变更时 |
| 架构设计文档 | 架构师 | 架构变更时 |
| 开发规范文档 | 全体开发者 | 规范优化时 |
| 进度追踪文档 | 项目负责人 | 每周 |
| 子项目文档 | 对应智能体 | 里程碑完成时 |

### 文档命名规范

- 使用中文命名，清晰表达文档内容
- 文档类型后缀：
  - 规划文档：`XX项目技术规划文档.md`
  - 报告文档：`XX项目XX报告.md`
  - 接口文档：`XXAPI接口文档.md`
  - 方案文档：`XX方案文档.md`
  - 会议记录：`会议记录.md` 或 `XX会议产出物.md`

### 文档版本控制

- 所有文档均使用 Git 进行版本控制
- 重要文档更新时，在文档开头标注：
  ```markdown
  > **最后更新**: 2026-01-14
  > **更新人**: [姓名]
  > **更新内容**: 简要说明更新内容
  ```
