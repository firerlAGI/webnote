# T2项目前端Web应用界面设计和视觉风格方案

## 1. 设计理念与原则

### 1.1 核心设计理念
- **极简设计**：减少视觉噪音，突出核心功能，创造清爽、专注的用户体验
- **一致性**：统一的视觉语言和交互模式，提高用户学习效率
- **功能性**：设计服务于功能，确保每个设计元素都有明确的目的
- **用户中心**：以用户需求为导向，优化操作流程和界面布局

### 1.2 设计原则
- **清晰性**：信息层次分明，视觉引导明确
- **简洁性**：去除冗余元素，保持界面干净整洁
- **可用性**：直观易懂的操作方式，减少学习成本
- **响应式**：适配不同屏幕尺寸，提供一致的跨设备体验
- **可访问性**：符合WCAG标准，确保所有用户都能正常使用

## 3. 字体系统

### 3.1 字体选择
- **主要字体**：Inter, -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, sans-serif
  - 特点：现代无衬线字体，清晰易读，适合屏幕显示
  - 优势：良好的跨平台支持，优秀的字重变化，适合构建层次结构

- **代码字体**：'Fira Code', 'Courier New', monospace
  - 特点：等宽字体，支持连字，适合代码显示
  - 优势：提高代码可读性，减少视觉疲劳

### 3.2 字体层次结构
| 级别 | 字体大小 | 字重 | 行高 | 用途 |
|------|---------|------|------|------|
| 标题1 | 24px | 700 | 1.2 | 页面主标题 |
| 标题2 | 20px | 600 | 1.2 |  section标题 |
| 标题3 | 18px | 600 | 1.3 | 子 section标题 |
| 标题4 | 16px | 600 | 1.3 | 卡片标题、模块标题 |
| 正文 | 14px | 400 | 1.5 | 主要文本内容 |
| 辅助文本 | 12px | 400 | 1.4 | 辅助信息、提示文本 |
| 小字 | 10px | 400 | 1.3 | 标签、状态指示 |

### 3.3 字体应用规范
- **标题**：使用较大字号和较重字重，建立清晰的视觉层次
- **正文**：使用适中字号和正常字重，确保良好的可读性
- **辅助文本**：使用较小字号和较轻字重，提供补充信息
- **字体一致性**：在整个应用中保持字体选择的一致性
- **响应式字体**：根据屏幕尺寸自动调整字体大小，确保在不同设备上的可读性

## 4. 布局系统

### 4.1 网格系统
- **基础单位**：4px
- **列数**：12列网格系统
- ** gutter宽度**：16px（桌面端），12px（平板端），8px（移动端）
- **最大容器宽度**：1440px，居中对齐

### 4.2 间距系统
| 间距名称 | 像素值 | 用途 |
|---------|-------|------|
| xs | 4px | 微小间距，如图标与文本之间 |
| sm | 8px | 小间距，如表单元素之间 |
| md | 16px | 中等间距，如组件之间 |
| lg | 24px | 大间距，如页面区域之间 |
| xl | 32px | 超大间距，如页面顶部和底部 |
| 2xl | 48px | 特大间距，如主要内容区域与边缘 |

### 4.3 布局结构
- **三栏布局**（桌面端）：
  - 左侧边栏：240px，包含导航菜单和文件夹列表
  - 主内容区：自适应，包含笔记列表或详情
  - 右侧边栏：300px，包含笔记信息、标签等（可选）

- **两栏布局**（平板端）：
  - 左侧边栏：200px（可折叠）
  - 主内容区：自适应

- **单栏布局**（移动端）：
  - 顶部导航栏：固定
  - 主内容区：自适应
  - 底部导航栏：固定，包含主要导航选项

### 4.4 布局组件
- **卡片**：独立的内容容器，包含相关信息和操作
- **面板**：可折叠的内容区域，用于组织相关功能
- **分割线**：用于分隔不同内容区域，提高视觉清晰度
- **空白区域**：战略性使用空白，增强内容可读性和视觉层次

## 5. 组件设计

### 5.1 按钮组件
- **样式变体**：
  - 主要按钮：主色调背景，白色文本
  - 次要按钮：白色背景，主色调边框和文本
  - 文本按钮：无背景，主色调文本
  - 危险按钮：红色背景，白色文本

- **尺寸变体**：
  - 大按钮：44px高度，适合主要操作
  - 中按钮：36px高度，适合常规操作
  - 小按钮：28px高度，适合次要操作
  - 图标按钮：32px × 32px，适合工具栏和紧凑区域

- **状态**：
  - 默认：标准样式
  - 悬停：轻微阴影和颜色变化
  - 激活：按下效果，颜色加深
  - 禁用：降低不透明度，禁止交互

- **圆角**：6px（中按钮），8px（大按钮），4px（小按钮）

### 5.2 输入框组件
- **样式**：
  - 标准输入框：白色背景，灰色边框
  - 聚焦状态：主色调边框，轻微阴影
  - 错误状态：红色边框，错误提示
  - 禁用状态：灰色背景，降低不透明度

- **尺寸**：
  - 标准输入框：36px高度
  - 文本域：可变高度，最小36px
  - 搜索框：36px高度，带有搜索图标

- **附加元素**：
  - 前缀图标：用于表示输入类型
  - 后缀图标：用于密码可见性切换、清除按钮等
  - 帮助文本：位于输入框下方，提供额外信息

### 5.3 卡片组件
- **样式**：
  - 标准卡片：白色背景，轻微阴影，灰色边框
  - 悬停状态：轻微上浮效果，阴影加深
  - 激活状态：主色调边框或指示器

- **尺寸**：
  - 标准卡片：自适应宽度，固定高度或可变高度
  - 笔记卡片：280px宽度，可变高度
  - 复盘卡片：自适应宽度，固定高度

- **内部布局**：
  - 卡片头部：标题、时间、状态等
  - 卡片内容：主要信息
  - 卡片底部：操作按钮、标签等

### 5.4 导航组件
- **侧边导航**：
  - 固定宽度：240px
  - 折叠状态：64px
  - 菜单项：图标 + 文本，悬停高亮
  - 活动项：主色调背景，白色文本

- **顶部导航**：
  - 高度：64px
  - 包含：logo、搜索框、用户信息、操作按钮
  - 响应式：在小屏幕上折叠为汉堡菜单

- **底部导航**：
  - 高度：56px
  - 包含：主要导航选项，图标 + 文本
  - 仅在移动端显示

### 5.5 对话框组件
- **样式**：
  - 模态对话框：居中显示，半透明背景遮罩
  - 非模态对话框：可拖动，不阻塞背景操作
  - 确认对话框：小型对话框，包含确认和取消按钮

- **尺寸**：
  - 标准对话框：最大宽度500px
  - 大型对话框：最大宽度800px
  - 小型对话框：最大宽度300px

- **布局**：
  - 对话框头部：标题、关闭按钮
  - 对话框内容：主要信息和表单
  - 对话框底部：操作按钮

### 5.6 通知组件
- **样式**：
  - 成功通知：绿色背景，白色文本
  - 错误通知：红色背景，白色文本
  - 警告通知：黄色背景，深色文本
  - 信息通知：蓝色背景，白色文本

- **位置**：
  - 顶部右侧：默认位置
  - 顶部中央：重要通知
  - 底部中央：移动端默认位置

- **动画**：
  - 滑入/滑出动画
  - 自动消失（可配置）
  - 点击关闭

### 5.7 加载组件
- **样式**：
  - 圆形加载器：旋转动画
  - 线性加载器：进度条动画
  - 骨架屏：内容占位符

- **应用场景**：
  - 页面加载：全屏加载器
  - 数据加载：局部加载器
  - 操作执行：按钮加载状态

- **动画**：
  - 平滑旋转或进度动画
  - 骨架屏渐显效果

## 7. 响应式设计策略

### 7.1 断点设置
- **移动端**：< 640px
- **平板端**：640px - 1024px
- **桌面端**：> 1024px
- **大屏桌面**：> 1440px

### 7.2 布局适配策略
- **移动端**：
  - 单栏布局
  - 底部导航栏
  - 折叠式侧边栏（通过汉堡菜单访问）
  - 简化的操作控件
  - 垂直排列的内容

- **平板端**：
  - 可折叠的两栏布局
  - 顶部导航栏
  - 适度简化的控件
  - 网格布局（2列）

- **桌面端**：
  - 三栏布局
  - 完整的侧边栏
  - 丰富的控件和选项
  - 网格布局（3-4列）

- **大屏桌面**：
  - 扩展的三栏布局
  - 更多的内容展示空间
  - 增强的视觉效果
  - 更大的网格布局（4-5列）

### 7.3 交互适配策略
- **触摸设备**：
  - 增大点击区域（至少44px × 44px）
  - 优化触摸手势（滑动、捏合缩放等）
  - 简化复杂操作
  - 提供明确的触摸反馈

- **鼠标设备**：
  - 精确的光标定位
  - 悬停效果和提示

## 8. 动效设计

### 8.1 核心动效原则
- **目的性**：动效服务于功能，增强用户理解
- **简洁性**：轻微、自然的动效，避免过度动画
- **一致性**：统一的动画风格和时间曲线
- **性能**：优化动画性能，避免卡顿

### 8.2 动效类型
- **过渡动画**：
  - 页面切换：平滑的淡入淡出或滑动效果
  - 元素显示/隐藏：轻微的淡入和缩放效果
  - 模态框：从中心淡入，背景轻微模糊

- **微交互**：
  - 按钮点击：轻微的按下效果
  - 表单验证：实时的输入反馈
  - 卡片悬停：轻微的上浮和阴影变化
  - 导航项选择：平滑的背景色过渡

- **状态指示**：
  - 加载状态：优雅的加载动画
  - 同步状态：微妙的同步指示器
  - 成功/错误：短暂的成功/错误提示动画

### 8.3 动画参数
- **持续时间**：
  - 微交互：100-200ms
  - 过渡动画：200-300ms
  - 复杂动画：300-500ms

- **缓动函数**：
  - 进入动画：ease-out（先快后慢）
  - 退出动画：ease-in（先慢后快）
  - 循环动画：linear（匀速）
