## 使用说明

### 如何使用此文档

1. **每日更新**：每天站会后立即更新此文档
2. **保持简洁**：每个智能体汇报2-3个重点即可
3. **问题跟踪**：阻塞问题要详细记录，包括影响和解决时间
4. **决策记录**：重要决策要记录负责人和截止日期
5. **风险提示**：提前识别风险，做好应对准备

### 站会流程

1. **准备阶段**（5分钟）
   - 查看自己负责的Issue列表
   - 更新Issue状态
   - 准备要汇报的内容

2. **站会进行**（15-30分钟）
   - 按顺序汇报：昨天完成、今天计划、遇到阻塞
   - 控制每人汇报时间在2-3分钟
   - 记录阻塞问题和决策

3. **会后整理**（5分钟）
   - 更新此文档
   - 更新GitHub Projects看板
   - 发送站会总结给团队

### 注意事项

- **准时参加**：站会准时开始，准时结束
- **聚焦问题**：讨论阻塞问题和解决方案
- **避免技术细节**：详细的技术问题会后单独讨论
- **行动明确**：每个决策都要有负责人和截止日期
- **持续改进**：定期回顾站会效果，优化流程
