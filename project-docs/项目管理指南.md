## 项目看板设置

### 第一步：创建GitHub Projects项目

1. 访问您的GitHub仓库：`https://github.com/firerlAGI/webnote`
2. 点击顶部的 **"Projects"** 标签
3. 点击 **"New Project"**
4. 项目信息：
   - **名称**: `极简网页笔记项目管理`
   - **模板**: 选择 **"Board"**（看板模板）
   - **可见性**: 选择 **"Private"**（私有）或 **"Public"**（公开）
5. 点击 **"Create Project"**

### 第二步：设置看板列

创建以下列（按从左到右的顺序）：

| 列名 | 说明 | 颜色 |
|------|------|------|
| 📋 待办 | 新任务、待开始的任务 | 灰色 |
| 🟡 进行中 | 正在处理的任务 | 黄色 |
| ✅ 已完成 | 已完成的任务 | 绿色 |
| ⚪ 阻塞 | 遇到阻塞的任务 | 红色 |

**设置步骤**：
1. 点击项目右上角的 **"..."** 菜单
2. 选择 **"Edit columns"**
3. 添加上述4列
4. 拖拽调整顺序

| 标签名 | 颜色 | 说明 |

| `priority:critical` | 🔴 红色 | 关键任务，必须立即完成 |


2. **选择对应模板**：
   - 使用 `type:feature` 标签

   - 分配负责人

   - 添加相关标签

## 自动化规则

### GitHub Projects 自动化

可以在Projects中设置以下自动化规则：

1. **当Issue关闭时**：
   - 自动移动到"✅ 已完成"列
   - 发送通知给负责人

2. **当Issue超过截止日期时**：
   - 添加 `overdue` 标签
   - 发送提醒通知

3. **当PR合并时**：
   - 自动关闭关联的Issue
   - 将Issue移动到"✅ 已完成"列

4. **当Issue被标记为"blocked"时**：
   - 自动移动到"⚪ 阻塞"列
   - 通知相关智能体

### GitHub Actions 自动化

创建 `.github/workflows/project-automation.yml`：

```yaml
name: Project Automation

on:
  issues:
    types: [opened, labeled, closed]
  pull_request:
    types: [opened, closed, merged]

jobs:
  issue-automation:
    runs-on: ubuntu-latest
    steps:
      - name: Comment on issue
        if: github.event_name == 'issues' && github.event.action == 'opened'
        uses: actions/github-script@v6
        with:
          script: |
            github.rest.issues.createComment({
              issue_number: context.issue.number,
              owner: context.repo.owner,
              repo: context.repo.repo,
              body: '👋 感谢创建Issue！请确保：\n1. 已添加正确的标签\n2. 已分配负责人\n3. 已设置截止日期'
            })
```
